# 每天家常菜应用完整开发计划

## 技术栈整合方案
- **前端框架**: React 18 + TypeScript + Vite
- **UI框架**: Tailwind CSS + shadcn/ui（移动端优先）
- **状态管理**: Zustand + React Query
- **后端服务**: Supabase（认证、数据库、存储、实时订阅）
- **部署平台**: Vercel（前端）+ Cloudflare（CDN加速和安全防护）
- **AI集成**: 百度AI API（图像识别、智能推荐）+ OpenAI API（菜谱生成）
- **版本控制**: GitHub
- **PWA**: Vite PWA Plugin（离线支持）

## 数据库设计（Supabase）

### 核心数据表
1. **users** - 用户信息表
   - 多渠道注册信息（微信、支付宝、手机号）
   - 所在城市、厨房设备
   - 用户偏好设置

2. **dishes** - 菜品表
   - 菜品名称、分类、食材、步骤
   - 营养信息、烹饪时间、难度
   - 图片、来源平台（下厨房/小红书/自建）

3. **weekly_plans** - 周计划表
   - 用户ID、日期、餐次（早/中/晚）
   - 菜品ID列表

4. **shopping_lists** - 采购清单表
   - 食材名称、数量、单位、预估价格
   - 储存方式、采购状态

5. **purchase_records** - 采购记录表
   - 时间、金额、类别、详细清单
   - 用于费用统计和可视化

6. **search_history** - 搜索历史表
   - 用户ID、搜索关键词、搜索时间

## 开发阶段规划

### 第一阶段：项目基础架构（2天）
1. 项目初始化
   - 创建React + TypeScript + Vite项目
   - 配置Tailwind CSS和shadcn/ui
   - 设置PWA配置和Service Worker
   - 配置React Router v6路由系统
   - 集成Supabase客户端

2. 开发环境搭建
   - 配置GitHub仓库
   - 设置Vercel部署流程
   - 配置Cloudflare CDN
   - 设置环境变量管理

### 第二阶段：用户系统（2天）
1. 多渠道注册登录
   - 手机号注册登录（短信验证码）
   - 微信登录（预留接口）
   - 支付宝登录（预留接口）
   - JWT token管理

2. 用户信息采集
   - 城市选择组件（支持搜索）
   - 厨房设备多选表单
   - 用户偏好设置页面

### 第三阶段：菜品数据系统（3天）
1. 数据获取与整合
   - 调研下厨房和小红书API可行性
   - 设计合规的数据爬取方案（遵守robots协议）
   - 建立数据清洗和标准化流程
   - 实现数据缓存机制（避免重复请求）

2. 自定义菜品搜索
   - 多维度搜索（关键词、分类、食材）
   - 高效搜索算法（全文索引）
   - 搜索历史记录
   - 热门搜索推荐
   - 搜索结果排序和筛选

### 第四阶段：AI推荐引擎（2天）
1. 百度AI集成
   - 菜品图像识别
   - 智能推荐算法
   - 营养分析功能

2. 菜谱推荐系统
   - 营养均衡算法（肉类、蛋类、蔬菜类）
   - 基于厨房设备的智能推荐
   - 基于地域的菜系推荐
   - 3秒响应优化策略

### 第五阶段：周菜谱规划（2天）
1. 周菜谱规划系统
   - 日历视图组件（周视图）
   - 早中晚餐时段选择
   - 拖拽式菜谱安排
   - 菜谱快速预览和编辑

2. 菜谱详情页面
   - 图文并茂的制作指南
   - 分步骤说明
   - 食材用量精确显示
   - 成品图片展示

### 第六阶段：采购管理系统（3天）
1. 智能采购表生成
   - 自动汇总食材清单
   - 智能去重和合并
   - 价格估算功能
   - 储存注意事项提示
   - 按采购地点分类

2. 采购费用数据汇总
   - 时间选择器（周视图/自定义时间段）
   - 费用统计与可视化（图表展示）
   - 总费用、分类费用占比
   - 多维度排序和筛选
   - 数据导出功能（CSV/Excel）

### 第七阶段：导航与分享（1天）
1. 标签页导航系统
   - "买菜"模块（采购清单）
   - "备菜"模块（食材准备）
   - "制作"模块（烹饪指导）

2. 微信分享功能
   - 生成分享卡片

### 第八阶段：优化与上线（2天）

   - 图片懒加载和压缩

   - 本地数据缓存（IndexedDB）
   - Service Worker离线支持
   - CDN加速优化

2. 数据安全与隐私
   - HTTPS强制加密
   - 敏感数据加密存储
   - 用户隐私权限管理
   - 符合国家数据安全法规
   - 完善的错误处理机制

3. 测试与部署
   - 单元测试（Vitest）
   - E2E测试（Playwright）
   - 移动端适配测试
   - 部署到Vercel
   - 配置Cloudflare CDN

